HTML

  • HTML = Hypertext Markup Language
  • HTML is one of three cornerstone languages used in web development. It is used to create the structure of a webpage.
  • HTML is made up of elements - the type of content on a webpage that can be seen by the user. Elements are also used to divide the content and describe the content to browsers and other developers. Typically these elements will not be visible to the user.
  • Elements use an opening and closing tag, represented by angle brackets.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element is an example of a semantic HTML element. Semantic HTML elements describe its content and the role it plays on a webpage. Using semantic HTML elements is important because they make the code easier to read. It also ensures the webpage is accessible to a wider variety of users, including those who use assistive technologies.
  • Non semantic elements do not tell us about the content they contain. Non semantic elements are still used when they are the best solution to a structural issue.
  • A footer element appears at the bottom of the webpage and normally contains information about the author, copyright, contact information, a sitemap of navigation information.
  • Elements that are nested inside other elements are called child elements. An element hosting child elements is known as the parent element.
  • An HTML attribute defines additional information about the element.
  • To create headings we use h1, h2, h3, h4, h5 and h6 tags. The content contained within the opening and closing tag is the heading. In an HTML file, we can only use h1 once.
  • We can use ul, ol and li elements to create unordered lists, ordered lists and list items.
  • We use the img element for images that will be displayed on the webpage. The img element can contain various attributes, including the src attribute. The src attitude defines the location of the image.
  • To set a line of empty space, or a line break, between two blocks of content we use a br element.
  • We can use an anchor element (shown as an to create links within the same webpage or to other webpages. A href attribute points us to the URL for the link.

CSS

  • CSS = Cascading Style Sheets
  • CSS is the language we use to style an HTML document and describes how HTML elements should be displayed.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheets.
  • Inline CSS is when a developer styles an element directly in the head section of the HTML file. Inline CSS is useful for quick and permanent changes, but less flexible than using internal and external CSS style sheets.
  • Internal CSS style sheet: an internal style sheet contains the CSS rules for the webpage using a style element in the head section of the HTML file. The CSS rules in this section will only apply to that webpage. Multiple elements on the webpage can be styled using CSS classes and IDs.
  • External CSS style sheet: CSS rules are coded in a separate file. A link to the external style sheet is placed using a link element in the HTML’s head section. This is normally the preferred method of styling HTML documents because design changes are easier to make.
  • Using an external CSS style sheet is an example of how we follow the principle of the separation of concerns. The separation of concerns is based on the idea that we should divide code into different files based on its use.
  • A CSS rule consists of a selector and a declaration block.
  • A selector points to the HTML element you want to style. We can combine multiple selectors into a singe CSS rule using a comma. This is a way we can write Do Not Repeat Yourself (DRY) code.
  • A declaration block contains two important components: the CSS property we want to apply (e.g. colour) and the value of that property (e.g. blue). We surround declaration blocks with curly braces.
  • Multiple CSS declarations are separated with semicolons.
  • CSS Box Model: a box wraps around every HTML element. It consists of margins, borders, padding and content.
  • A margin indicates how much space we want around the outside of an element and the border.
  • A padding indicates how much space we want around the content inside an element.
  • DRY code: the idea that developers can reduce the amount of code that is written by reusing or sharing pieces of code. This reduces the overall size and the complexity of the code.

Git

  • git status: checks what branch we are currently working on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds changed files in our working directory to the ‘staging’ area of git
  • git commit -m “what has been changed”: committing the staged changes to our git repository
